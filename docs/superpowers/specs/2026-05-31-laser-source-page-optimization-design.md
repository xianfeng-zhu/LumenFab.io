# Design: 激光源页面内容优化

## 目标

重构 `src/pages/components/laser-source.mdx` 的结构、段落组织和用词，以 PIC 页面 (`pic.mdx`) 的组织模式为参考，使页面更易读、逻辑更清晰，同时保持现有技术深度。

## 范围

- **仅修改** `src/pages/components/laser-source.mdx`
- 不动其他页面，保持现有交叉引用链接有效
- 保留所有 TermNote 组件的 term/note 内容
- 保留 DfbLaserModel 3D 模型组件，不修改其代码
- 模型位置前移：参考 PIC 页面将 PicChipModel 放在导语之后、正文之前的方式，将 DfbLaserModel 从当前位置（工作原理之后）移到 §0 导语和 §1 结构地图之间

## 目标读者

兼顾两类读者：需要建立整体认知框架的入门学习者，和需要技术深度的工程师。页面应支持「先快速扫描获取框架，再细读感兴趣的部分」的阅读模式。

## Wording 原则

- 去重：合并跨段落重复的相似表述
- 断句：拆分超过 40 字的中文长句
- 统一：同一概念在全文中使用固定术语和表述方式
- 凝练：删除「可以理解为」「换句话来说」等填充词，除非用于引入真正的新视角
- 正式：用词正式、自然，使用规范和通用的中文技术术语，避免生硬的翻译腔（如英文直译句式、不自然的连接词、过度西化的长定语）
- 直陈：尽量少用「不是……而是……」等否定-肯定对照结构；直接陈述事实本身，读者不需要先排除一个错误答案再获知正确答案
- 保留：技术精度不打折，物理因果关系不简化

## 各节详细设计

### §0. 什么是 InP / III-V 半导体激光源（新增，替代现有导语 + CPO 部分）

**结构：**

1. **一句话定义**（3-4 句）：InP/III-V 边发射 DFB 激光器是什么，在光引擎中扮演什么角色
2. **在系统中的位置**（简洁文本链）：
   ```
   激光源 → 连续波光 → 耦合器 → PIC → 调制 → 光纤
   ```
3. **约束传递表**（新增，参考 PIC §0 的激光器-PIC 接口表）：激光器参数 → 约束 PIC/系统什么决策

   | 激光器参数 | 下游受影响的决策 |
   |-----------|-----------------|
   | 中心波长 | WDM 通道匹配、微环/AWG 设计目标 |
   | 输出功率 | 链路功率预算、PIC 分光比 |
   | RIN | 接收端 SNR、调制消光比需求 |
   | SMSR | WDM 邻道串扰 |
   | 线宽 | 相干通信可用调制格式 |
   | 温漂 | 热调谐范围、TEC 需求 |
   | 老化漂移 | 5-10 年链路预算余量 |
   | 偏振 / PER | 耦合器选型、偏振分集需求 |
   | 模场尺寸 | 边缘耦合 vs 光栅耦合 |

4. **CPO 为什么提高要求**（压缩现有内容为 4-5 条要点列表 + 一小段收束）
5. **阅读索引表**（新增，参考 PIC §1 的分类地图）：

   | 章节 | 内容 | 适合谁 |
   |------|------|--------|
   | §1 | 结构分类地图 | 先看这个建立全局认知 |
   | §2 | 工作原理 | 理解物理机制 |
   | §3–8 | 制造路径 | 深入各阶段工艺 |
   | §9 | 评价指标 | 工程师选型参考 |
   | §10 | 从芯片到系统 | 理解下游约束 |

**用词注意：**
- 导语从现有 3 段密集文字压缩为简洁的结构化入口
- 「DFB / CW / CW-DFB」定义保留但移到更自然的位置（出现在 §0 的定义句中，不再单独一段）

### §1. 激光器里有什么：结构分类地图（新增）

**结构：**

一张表，类似 PIC §1 的器件分类地图：

| 结构类别 | 包含什么 | 在激光器里的作用 |
|---------|---------|----------------|
| 衬底 | InP 单晶晶圆 | 晶格模板、机械支撑、下侧导电 |
| 外延层 | 包层、SCH、量子阱有源区、接触层 | 产生光增益、垂直光场限制、电流注入 |
| 波导 | Ridge waveguide / buried heterostructure | 横向光场与电流约束 |
| 光栅 | DFB 光栅、相移结构 | 波长选择、单模稳定 |
| 电极与绝缘 | p/n 接触、介质钝化、电流窗口 | 电流注入路径、漏电控制 |
| 端面 | 解理面、AR/HR 镀膜 | 输出耦合、边界反射控制 |

每类结构标注对应详细章节号。

**用词注意：**
- 简洁的功能描述，不展开物理原理（留给 §2–§8）
- 这张表的作用是「地图」，不是「百科全书」

| 子节 | 固定标题模式 | 内容 |
|------|------------|------|
| X.1 | 做什么 | 物理/工程定义，这一步在激光器里形成什么结构 |
| X.2 | 怎么做 | 工艺路径、关键设备、控制参数 |
| X.3 | 对器件的影响 | 对比表：工艺变量 → 物理影响 → 器件表现 |
| X.4 | 量产壁垒 | 为什么难复制，成熟厂商的优势来自哪里 |
