## EIC 是光引擎的电子边界

<TermNote label="EIC" note="electrical integrated circuit，电子集成电路。CPO 光引擎中，它通常指靠近 PIC 的高速电子芯片或电子功能区，负责 driver、TIA、时钟、均衡、控制和部分电源管理。" /> 把 switch ASIC 或 host SerDes 的高速电信号送到 PIC，也把 photodetector 产生的微弱电流读回电域。它的核心问题是系统分工：哪些补偿和判决留在 host，哪些放进光引擎，哪些由 PIC / EIC / package 共同决定。

EIC 与 [PIC / optical circuit](../pic/) 共用高速边界：driver 面向 modulator electrode，TIA 面向 photodetector。laser bias 与光源管理见 [激光源](../laser-source/)；fiber array 和 coupler 见 [Optical I/O / fiber coupling](../optical-io/)；EIC、PIC、ASIC 与 substrate 的距离、热和返修问题见 [Package integration](../packaging/)；KGD、封装后测试和 burn-in 见 [Manufacturing and test](../manufacturing-test/)。

相关基础页：[Electrical signal to optical signal](../../learn/electrical-signal-to-optical-signal/)、[Tx path and Rx path](../../learn/tx-rx-path/)、[Inside a transceiver](../../learn/inside-a-transceiver/)、[Bandwidth density and power per bit](../../learn/bandwidth-density-and-power-per-bit/)。

## 先决定系统分工：retimed、linear 与 direct-drive

CPO 的 EIC 形态由 host 和 optical engine 的分工决定。光引擎可以集成完整 CDR / DSP，也可以只保留高线性模拟前端，还可以把 driver 或 receiver 进一步推向 direct-drive co-design。

| 边界类型 | Engine 内 EIC 功能 | 系统含义 |
|---|---|---|
| Retimed | CDR / retimer，可能包含 DSP、ADC / DAC、diagnostics 和完整 PHY 功能 | 接口清晰，局部补偿强；功耗、面积、热和测试复杂度上升 |
| Linear amplified | driver、TIA 和必要模拟放大，减少本地 regeneration | engine 功耗和延迟有机会下降；host SerDes 承担更完整补偿 |
| Half-retimed | 一个方向 retimed，另一个方向 linear | 按 Tx / Rx 难度拆分补偿责任 |
| Direct drive | ASIC 或 host I/O 直接面对调制器或 PD / TIA 边界 | 电路径极短；封装寄生、线性度、调试和测试访问要求很高 |

减少 engine 内 DSP 并不会让信道损伤消失。均衡、判决、jitter tolerance、PAM4 linearity 和 FEC margin 仍要由 host SerDes、PIC、电极、PD、package channel 和 calibration 共同承担。EIC 页面应先读这一层分工，再读 driver、TIA 和控制电路。

## Tx 前端：driver 与调制器共同定义

Driver 的任务是在目标 baud rate 下给 PIC 调制器提供足够快、足够准、足够稳定的电压或电流波形。它是面向具体 modulator load 的高速模拟前端。

| 调制器或负载 | Driver 关注点 | 典型风险 |
|---|---|---|
| Silicon MZM | 差分摆幅、行波电极阻抗、速度匹配、VpiL、RF 损耗 | 摆幅不足、速度失配或电极损耗导致 optical eye 闭合 |
| Microring modulator | 小电容、谐振工作点、PAM4 线性、bias / heater 协同 | 温漂和非线性造成电平不均或锁定范围不足 |
| EAM / EML | 反向偏置、电吸收曲线、chirp、光功率承受 | 偏置窗口和吸收非线性影响 ER、TDECQ 和寿命 |
| TFLN MZM | 高速电极、封装走线、较低光损耗、驱动摆幅 | 封装电损耗、驱动功耗和热路径共同限制带宽 |

PAM4 让 driver 线性度、抖动和电平控制变得关键。driver 输出电平、调制器转换曲线和光功率响应会叠加成 optical eye；电极、电容、microbump、interposer trace 和 termination 会共同形成频率响应。工程上常用 peaking、pre-emphasis、FFE、segmented driver、bias DAC、calibration loop 或数字辅助预失真来保护 OMA、ER、TDECQ 和 BER。

## Rx 前端：PD-to-TIA 是接收端第一段链路

Photodetector 输出的是电流，<TermNote label="TIA" note="transimpedance amplifier，跨阻放大器。它把 photodiode 输出电流转换成电压，并在带宽、噪声、增益和饱和之间取舍。" /> 把这个电流转换成后级可以处理的电压。简化关系是：

```text
Vout ≈ Ipd × transimpedance gain
```

接收端难点在于这个关系要在几十 GHz 带宽、很小输入电流、PD 电容、pad、bump、ESD 和封装寄生下成立。

| 取舍 | 含义 | 对链路的影响 |
|---|---|---|
| gain vs. bandwidth | 更高跨阻有利于小信号灵敏度，也更容易受输入电容和稳定性限制 | 影响 receiver sensitivity 和可达 baud rate |
| noise vs. power | 低输入等效噪声通常需要更强输入级和更高偏置电流 | 影响灵敏度，同时进入 engine 热预算 |
| linear range vs. sensitivity | 高增益容易饱和，低增益会提高小信号噪声压力 | 影响 PAM4 动态范围和接收余量 |
| PD capacitance vs. bandwidth | PD 结电容、pad 和互连寄生形成输入极点 | 影响高频响应、TIA stability 和封装布局 |

高速 data PD 与 monitor PD 的读出目标不同。data PD 追求带宽、噪声和动态范围；monitor PD 追求稳定、线性和低漂移。EIC 往往同时包含高速 TIA 和低速 monitor readout，两者共享电源、地、温度和管理接口，但指标体系不同。

## SerDes、CDR 与 DSP：补偿责任放在哪里

<TermNote label="CDR" note="clock and data recovery，时钟数据恢复。它从接收数据中恢复采样时钟，并把数据重新定时，降低积累抖动对后级的影响。" /> 解决采样时序问题。<TermNote label="DSP" note="digital signal processor，数字信号处理器。光互连中常用于均衡、预失真、判决反馈、FEC 辅助、非线性补偿和链路诊断。" /> 解决波形和通道损伤问题。CPO 把电路径缩短后，CDR / DSP 的位置仍然是系统架构选择。

| 边界 | Host / switch ASIC 侧 | 光引擎 EIC 侧 |
|---|---|---|
| DSP-based optical PHY | 负责系统侧 SerDes 和协议接口 | 集成光侧 DSP、driver、TIA、CDR、diagnostics |
| Short-reach / VSR CPO interface | 负责短距离高速电互连和部分 margin | 负责靠近 PIC 的 driver / TIA 和必要 retiming |
| Linear interface | Host SerDes 承担完整均衡和判决 | 保留高线性 driver、TIA 和必要模拟增益 |
| Direct-drive co-design | ASIC I/O 面向光电器件条件优化 | EIC 极简或成为专用 analog front end |

这一层分工会改变功耗和可诊断性。Retimed EIC 能在 engine 内部看见更多链路状态，也带来更多热和固件复杂度；linear EIC 功耗潜力较好，但需要 host 看到并补偿更长的模拟链路；direct-drive 需要 PIC / EIC / ASIC / package 同时建模。

## 电源、时钟、控制：让 PIC 长期停在正确工作点

EIC 的低速电路常决定光引擎能否稳定量产和运维。Driver、TIA、PLL、ADC / DAC、heater control、monitor readout、I2C / SPI / CMIS management、BIST、PRBS 和 alarm 都要在狭小封装里共存。

| 功能 | 主要内容 | 为什么重要 |
|---|---|---|
| power integrity | analog / digital / driver / TIA / PLL / heater rails、decap、IR drop、ground bounce | 电源噪声会进入 jitter、driver swing、TIA noise 和控制读数 |
| clocking | reference clock、PLL、CDR、phase interpolator、lane deskew | 决定采样窗口、lane 对齐和 jitter tolerance |
| PIC control | DAC、ADC、monitor PD readout、temperature sensor、heater / bias loop | 保持 MRM resonance、MZM bias、WDM channel 和 optical power |
| diagnostics | PRBS、scan、BIST、loopback、alarm coverage、telemetry | 支持 wafer test、bring-up、failure isolation 和现场维护 |

控制环路的价值在长期运行中最明显。CPO optical engine 难以像前面板模块那样直接更换，EIC 的 telemetry 粒度、校准流程和故障定位能力会直接影响运营成本。

## PIC / EIC / 封装的共同设计条件

EIC 指标在封装后才算真正成立。driver 到 modulator 的路径、PD 到 TIA 的路径、power delivery、thermal map 和 test access 需要与 PIC 和 package 同时定义。

| EIC 条件 | 传给 PIC / 封装的影响 |
|---|---|
| driver output swing / impedance | 决定 modulator electrode、termination、microbump 数量和 RF routing |
| driver-to-modulator path length | 决定 interposer / substrate 走线、return path、reflection、loss 和 skew |
| TIA input capacitance budget | 决定 PD 面积、pad、bump、ESD 和 placement |
| PD-to-TIA placement | 决定接收噪声、带宽、封装布局和可测试性 |
| rail count / peak current | 决定 package power delivery、decap、VRM 位置和热点 |
| clock and lane deskew | 决定 reference clock routing、jitter isolation 和多 lane 同步 |
| ADC / DAC and sensors | 决定 PIC heater、bias、monitor tap 和 WDM control 的可实现性 |
| test access | 决定 wafer probe、package probe、loopback、PRBS、scan 和 failure analysis 覆盖 |

这些条件会反向改变 EIC 架构。若 modulator 电容较大，driver 需要更强输出或更短互连；若 PD 电容较大，TIA 噪声和带宽承压；若 package substrate 提供的 return path 不连续，实验室里的 driver / TIA 指标会在系统中下降。

## 测试、bring-up 与可诊断性

EIC 的测试不能停在 DC pass / fail。CPO 需要把电学测试、光电联合测试和管理状态串起来。

| 阶段 | 重点证据 | 作用 |
|---|---|---|
| wafer / die test | leakage、scan、PLL lock、driver swing、TIA gain、ADC / DAC、temperature sensor | 建立 EIC KGD 和分档 |
| subassembly test | bump continuity、RF path、power rail、control loop access | 确认封装动作没有破坏高速和管理路径 |
| package bring-up | PRBS、eye、BER、TDECQ、lane skew、thermal sweep、calibration log | 验证 EIC 与 PIC、laser、fiber 和散热路径共同工作 |
| field diagnostics | telemetry、alarm、FEC counters、loopback、firmware state | 支持失效定位、预防性维护和降级运行 |

可诊断性需要提前设计。封装后发现 BER 上升时，系统要能区分 driver swing、modulator bias、laser power、coupler loss、TIA noise、PD responsivity、power rail 或 clock jitter。缺少 monitor、loopback 和状态记录时，故障定位会变成昂贵返厂分析。

## 判断 EIC 的核心指标

EIC 指标要按功能分组。最大 lane rate 或工艺节点只能说明局部能力，不能证明它适合某个 optical engine。

| 指标组 | 典型参数 | 工程含义 |
|---|---|---|
| Tx driver | swing、bandwidth、rise / fall time、jitter、linearity、pre-emphasis、output impedance | 决定 modulator 能否形成足够 OMA、ER 和低 TDECQ |
| Rx TIA | transimpedance gain、input-referred noise、bandwidth、saturation current、linearity | 决定 receiver sensitivity、dynamic range 和 PAM4 接收能力 |
| SerDes / DSP / CDR | lane rate、FFE / DFE / CTLE taps、CDR jitter tolerance、BER margin | 决定补偿责任和接口清晰度 |
| power / thermal | pJ/bit、rail count、peak current、junction temperature、sensor coverage | 决定功耗密度、散热路径和长期稳定性 |
| package interface | bump pitch、pad map、ESD、trace length、PD-to-TIA parasitics | 决定电路指标能否在封装后保持 |
| control / telemetry | ADC / DAC resolution、calibration time、alarm coverage、PRBS / BIST | 决定 bring-up、调试、运维和可靠性管理 |

## 总结：EIC 是系统分工层

EIC 是 CPO 光引擎的电子分工层。它在 Tx 侧把 SerDes 信号变成适合调制器的高速波形，在 Rx 侧把 PD 电流变成可恢复的数据电压，并通过 clock、power、control 和 telemetry 维持 PIC 的工作点。

理解 EIC，要同时看系统接口和物理接口。系统接口决定 retimed、linear 或 direct-drive；物理接口决定 driver-to-modulator、PD-to-TIA、power delivery、thermal coupling 和 test access。只有这两层同时闭合，EIC 才能把 CPO 的短电路径收益变成稳定产品。
