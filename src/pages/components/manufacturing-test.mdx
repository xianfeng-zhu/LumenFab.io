## Manufacturing and test 贯穿整个制造链

Manufacturing and test 是 CPO 光引擎的制造控制系统。它跨越 laser、PIC、EIC、optical I/O、package 和链路级行为，判断哪些 die 可以进入封装，哪些组装结果可以继续流转，哪些 package 可以成为可交付产品。

这条链路覆盖多个测试插入点：

```text
incoming wafer / material inspection
→ wafer-level optical / electrical test
→ die sort and known-good-die
→ die attach / flip-chip / fiber attach inspection
→ assembly optical / electrical test
→ package-level electro-optic test
→ link-level eye / TDECQ / BER / FEC margin
→ aging, burn-in, stress screen
→ yield analysis and process feedback
```

激光器芯片级测试见 [激光源](../laser-source/)；片上光路、调制器和 PD 属于 [PIC / optical circuit](../pic/)；driver、TIA、SerDes 和控制电路属于 [EIC / driver and receiver](../eic/)；fiber array、coupler 和 connector 属于 [Optical I/O / fiber coupling](../optical-io/)；多 die 布局、散热和返修边界属于 [Package integration](../packaging/)。

相关基础页：[Inside a transceiver](../../learn/inside-a-transceiver/)、[Tx path and Rx path](../../learn/tx-rx-path/)、[Why photonic packaging is hard](../../learn/why-photonic-packaging-is-hard/) 和 [Why semiconductor lasers are temperature-sensitive](../../learn/why-semiconductor-lasers-are-temperature-sensitive/)。

## 质量门：每一步在更高成本前拦住什么风险

测试插入点的核心逻辑是“越早越便宜，越晚越真实”。早期测试靠近制造源头，适合建立 wafer map 和 die sort；后期测试更接近真实系统，能看到封装、光耦合、供电、热和控制环路共同作用。

| 质量门 | 拦截风险 | 测试证据 | 决策输出 |
|---|---|---|---|
| Incoming / process metrology | 材料、外延、SOI / SiPh wafer、金属和封装材料异常 | 厚度、应力、颗粒、片阻、反射谱、显微 / SEM / X-ray | 批次是否进入后续加工 |
| Wafer-level test | 工艺漂移、坏区、片内均匀性不足 | DC / RF / optical probe、L-I-V、插损、光谱、S-parameter、BIST | wafer map、die map、初筛 |
| Die sort / KGD | 坏 die 被贴进高价值封装 | pass / fail、binning、温度点、早期可靠性筛选 | KGD 清单和分档规则 |
| Assembly test | 贴装、互连、光纤对准、清洁和热路径偏离窗口 | AOI、X-ray、continuity、coupling loss、alignment drift、thermal path | 半成品继续流转、返工或报废 |
| Package-level link test | 多 die、多 lane、多温度条件下链路不闭合 | optical power、OMA、eye、TDECQ、BER、FEC margin、温扫 | package 放行、分档或失效分析 |
| Aging / stress screen | 早期失效和漂移速度过快 | burn-in、HTOL、thermal cycling、humidity、vibration、ESD | 批次风险、寿命模型和维护假设 |

质量门的目的，是在最终测试前尽量拦住可提前识别的问题。CPO 中的 package 价值高、返修窗口小，多 die 和高 lane count 会放大后段发现问题的成本。

## 测试插入点：从 wafer 到 package 的证据链

每个插入点看到的物理对象不同。Wafer-level test 能看到工艺分布；die sort 把连续参数变成封装输入；assembly test 看到贴装和对准是否改变窗口；package-level test 看到真实链路；aging 看到时间维度。

Wafer-level test 的主要产物是 wafer map。对 PIC，它可以显示 propagation loss、coupler loss、ring resonance、AWG passband、modulator bandwidth、PD responsivity 和 heater power 的空间分布。对 EIC，它可以显示 leakage、scan、PLL lock、driver swing、TIA gain、ADC / DAC 和 power rail。对 laser wafer 或 bar，完整光输出、端面镀膜、L-I-V、光谱和 burn-in 往往要到 cleaved bar、chip、COS 或封装阶段才能可靠测量。

Assembly test 关注封装动作是否破坏窗口。die attach 影响空洞、热阻和 tilt；flip-chip 影响高速寄生、open / short 和 bump fatigue；fiber attach 影响 coupling loss、return loss、偏振、胶水固化和清洁度；underfill 和 lid attach 影响应力、湿气阻挡、热路径和返修。

Package-level test 把对象从器件变成链路。此时要测 optical power budget、OMA、eye diagram、TDECQ、BER、FEC margin、lane skew、crosstalk、calibration loop、功耗和温度扫描。

## 成本阶梯：为什么 KGD、die sort 和 assembly test 必须前置

<TermNote label="KGD" note="Known-Good-Die，已知合格芯片。它表示裸 die 在进入更高层级组装前已经通过约定测试和筛选；KGD 是基于可测项目的置信状态，后续封装仍可能引入新的失效。" /> 的价值来自成本阶梯。裸 die 发现问题，损失相对有限；封进 optical engine 后发现问题，会牵连 PIC、EIC、laser、fiber array、substrate、散热结构和测试时间；与高价值 ASIC 深度结合后，返修成本进一步上升。

多 die package 的良率可以用简化模型理解：

```text
Y_package ≈ Y_1 × Y_2 × ... × Y_N
```

当关键 die 或关键光学接口数量变多时，单点良率的小幅下降会被乘法放大。真实制造中还存在相关性：同一 wafer 区域、同一 assembly 工位、同一 fiber attach 批次或同一热路径问题，会让失效按组出现。KGD、die sort 和 assembly test 的目标，是在更高成本前识别这些风险，并把它们变成可管理的 bin、hold、rework 或 scrap 决策。

| KGD 信息 | 目的 | 例子 |
|---|---|---|
| 基本可用性 | 排除明显坏片 | open / short、leakage、pad damage、PD 暗电流异常、laser 无法达到目标功率 |
| 参数分档 | 把连续参数放入可用窗口 | laser wavelength bin、PIC insertion-loss bin、modulator bandwidth bin、EIC speed / power bin |
| 风险标记 | 给后续封装和可靠性使用 | burn-in drift、wafer edge、温度窗口偏窄、alignment margin 偏小、test uncertainty 偏高 |

## 测试覆盖：哪些风险早测，哪些风险后测

测试覆盖要按风险组织。很多指标可以早测，但最终确认要等到封装或链路级条件出现。

| 风险类型 | 最早可测位置 | 最终确认位置 | 覆盖盲区 |
|---|---|---|---|
| PIC 插损和光谱漂移 | wafer-level optical test | package-level power budget / temperature sweep | fiber attach、package stress 和真实热场 |
| Modulator / driver 匹配 | wafer / die-level S-parameter、EIC test | package eye / TDECQ / BER | microbump、substrate 和控制环路耦合 |
| PD / TIA 接收余量 | PD wafer test、TIA die test | package receiver sensitivity / BER | PD-to-TIA 寄生和封装噪声 |
| Laser 输出与老化 | chip / bar / COS test、burn-in | package optical power / operations telemetry | 耦合、热路径和外置供光连接器 |
| Optical I/O 对准 | assembly coupling test | thermal cycling 后 loss / return loss | 胶水老化、污染、插拔和现场弯折 |
| System link quality | package-level test | traffic run、FEC counters、field telemetry | 长期漂移、固件状态和网络调度 |

这种覆盖矩阵能帮助团队决定哪些测试应该前置，哪些必须保留到最终链路。比如 PIC 插损可以在 wafer 上早测，但 fiber attach 漂移要到 assembly / thermal cycling 才能确认；TDECQ 和 BER 只能在 package-level 或 link-level 才接近真实。

## 数据闭环：wafer map、assembly log 和 package BER 怎样关联

测试只有变成反馈，才会成为制造能力。单次 pass / fail 可以放行产品；长期数据能告诉团队该收紧哪一个窗口。

| 数据 | 能回答的问题 |
|---|---|
| Wafer map | 异常是否与晶圆位置、曝光场、刻蚀区域、外延均匀性、金属片阻或 coupler 版图相关 |
| Die sort bin | 哪些参数连续漂移，哪些 die 适合低规格，哪些必须报废 |
| Assembly log | 异常是否与贴装设备、批次、胶量、固化曲线、清洁度、对准工位相关 |
| Package test | BER、TDECQ、功耗、热图和 lane balance 能否追溯到上游 die 或 assembly 记录 |
| Aging result | 漂移和早期失效是否集中在某个 wafer、bar、批次、封装材料或温度区间 |
| Failure analysis | 失效是否来自端面、波导、PD、bump、underfill、fiber attach、污染、ESD 或热应力 |

良率反馈会形成筛选规则调整、工艺窗口调整和设计改进。比如收紧 wavelength bin、提高 insertion-loss margin、改变 fiber attach 固化曲线、增加 monitor tap、扩大 heater tuning range 或调整 package thermal path。CPO 量产能力很大程度取决于这些数据能否快速关联。

## 量产决策：放行、分档、返工、报废和工程变更

测试数据要转成制造决策。常见决策包括：

| 决策 | 适用情形 | 风险 |
|---|---|---|
| 放行 | 指标在目标窗口内，趋势和批次记录正常 | 测试覆盖不足会放过潜在问题 |
| 分档 | 连续参数偏离最佳值，但可满足较低规格或特定配置 | 分档规则过宽会转移到后段失效 |
| 返工 | 连接器清洁、fiber attach、socket 接触或某些装配问题可恢复 | 返工可能引入新污染、新应力或追溯复杂度 |
| 报废 | 关键 die、光路、封装或可靠性风险不可恢复 | 需要把损失反馈到前段筛选 |
| 工程变更 | 问题呈批次、设计或工艺相关性 | 要更新 DFM、test limit、process window 或 design rule |

制造测试的组织能力体现在决策一致性。不同测试站点、不同批次和不同供应商的数据格式要能汇合；否则 pass / fail 只是局部结论，难以支撑 CPO 的高价值封装。

## CPO 的特殊难点

CPO 测试压力来自四个方面。高 lane count 让尾部分布变得重要；多 die 集成让组合良率变得敏感；光耦合让测试夹具和清洁状态成为测量的一部分；靠近 ASIC 的热环境让室温单点测试不能代表系统运行。

这些难点共同要求测试体系具备三种能力：早期质量门，覆盖矩阵，数据闭环。早期质量门降低后段成本；覆盖矩阵说明哪些风险已被看见；数据闭环把测试结果回到材料、工艺、版图、装配和可靠性模型。

## 总结：制造测试把 demo 变成可交付产品

Manufacturing and test 把 CPO 光引擎从可演示链路变成可交付产品。它由质量门、成本控制、覆盖设计和数据反馈共同组成。
