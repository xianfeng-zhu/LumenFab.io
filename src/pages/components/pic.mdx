## PIC 是光引擎的片上光路

<TermNote label="PIC" note="photonic integrated circuit，光子集成电路。它把波导、分光器、调制器、滤波器、耦合器和探测器等光学功能做成芯片级电路。" /> 是光引擎里的 optical circuit。连续光进入 PIC 后，会被分配到多条 lane，被调制器写入数据，再通过 MUX / DEMUX、monitor tap、coupler 或 photodetector 形成可接入光纤和 EIC 的链路。

PIC 页面关注的是“光怎样在芯片里被组织成电路”。激光源负责产生连续光，见 [激光源](../laser-source/)；driver、TIA 和控制电路属于 [EIC / driver and receiver](../eic/)；edge coupler、fiber array 和 connector 属于 [Optical I/O / fiber coupling](../optical-io/)；die 摆放、散热、返修和封装级良率属于 [Package integration](../packaging/)。

## 先看架构：Tx、Rx、PSM 与 WDM

PIC 的架构先回答三件事：光从哪里进来，数据在哪里写入，多少条 lane 通过多少根光纤离开。Tx PIC 通常从 CW laser input 开始，经过 splitter / power distribution、modulator、WDM MUX 或 parallel routing，再到输出 coupler。Rx PIC 则从 fiber input 开始，经过 coupler、DEMUX 或 parallel routing，最后到 photodetector 和 TIA。

```text
Tx: CW light -> splitter -> modulator array -> MUX or parallel routing -> optical I/O
Rx: optical I/O -> DEMUX or parallel routing -> photodetector array -> EIC TIA
```

PSM / parallel optics 把多条 lane 分别送入多根光纤，光学复用压力较小，fiber count 和 connector density 压力较大。WDM 把不同波长合到一根光纤，提高单纤带宽密度，同时把 laser wavelength、filter center wavelength、thermal drift 和 channel crosstalk 放进同一个预算。

| 架构选择 | 收益 | 主要约束 |
|---|---|---|
| PSM / parallel optics | 光路直观，滤波压力小，调试路径清楚 | fiber 数量、connector 体积、fiber escape 和多通道装配 |
| WDM | 单纤带宽密度高，适合高 lane count | 波长管理、AWG / ring 漂移、串扰、heater power 和监控 |
| Tx-heavy PIC | 调制、复用、monitor 和输出耦合为中心 | laser power budget、driver-to-modulator 路径和热调谐 |
| Rx-heavy PIC | DEMUX、photodetector 和 monitor 为中心 | PD responsivity、PD-to-TIA 寄生、接收灵敏度和噪声 |

这种架构视角比单独列器件更重要。splitter、MZM、MRM、AWG、PD 和 coupler 的指标都要回到同一个问题：这张片上光路能否在目标 lane rate、目标 fiber 数量、目标功耗和目标封装窗口内闭合。

## 平台决定器件库边界

SiPh、InP、SiN、TFLN / LNOI 是材料或平台路线；MZM、MRM、EAM、AWG、coupler、PD 是器件或结构。理解 PIC 时要把这两个层级分开。平台决定哪些器件容易做，器件库决定设计者能用哪些 building blocks 组成电路。

| 平台 | 适合承担的功能 | 需要协同的边界 |
|---|---|---|
| Si / SOI / SiPh | 紧凑波导、splitter、MZM / MRM、Ge PD、monitor、heater、复杂 PDK | 高性能通信光源通常需要 III-V 光源或异质集成 |
| InP | laser、SOA、EAM、部分 MZM、PD、有源 PIC | 无源复杂电路、低成本大规模版图和 silicon ecosystem |
| SiN | 低损耗 routing、filter、AWG、external cavity、interposer | 原生增益、高速有源调制和探测 |
| TFLN / LNOI | 高速、低损耗、线性电光调制 | 光源、探测、复杂封装和大规模集成生态 |

平台选择会改变后续所有设计问题。SiPh 的优势来自紧凑集成、foundry / PDK 和复杂版图；InP 的优势来自直接带隙和有源器件；SiN 适合低损耗路径和滤波；TFLN 的 Pockels 效应适合高性能调制。真实光引擎常把这些能力组合起来，让不同平台分别承担擅长的功能。

## 片上光路怎样分配损耗、相位和波长

PIC 的主预算是整张光路的损耗、相位和波长分配。输入光进入芯片后，会经历输入耦合、分光、波导传播、调制器插损、MUX / DEMUX、tap monitor 和输出耦合。每一处损耗都会反向抬高 laser power、热负载和可靠性压力。

| 光路单元 | 控制变量 | 下游后果 |
|---|---|---|
| waveguide / bend | propagation loss、bend radius、sidewall roughness、polarization behavior | 影响总插损、面积、串扰和版图密度 |
| splitter / MMI / directional coupler | splitting ratio、excess loss、bandwidth | 影响 lane-to-lane power balance 和 laser power margin |
| AWG / ring / Echelle | passband、center wavelength、channel spacing、crosstalk | 决定 WDM channel 是否对齐以及温控范围 |
| tap monitor | tap ratio、monitor linearity、layout location | 决定控制环路看到的是主光路真实状态还是局部代理信号 |
| coupler interface | mode size、pitch、PDL、return loss | 交给 optical I/O 和 package 的装配条件 |

相位和波长同样是预算。MZM 依靠两臂相位差形成强度调制，ring filter 依靠 resonance 选择波长，AWG 依靠波导阵列的相位差分离通道。温度、工艺偏差和封装应力都会改变有效折射率，进而改变这些相位关系。CPO 把 PIC 放到 ASIC 附近后，heater power、thermal crosstalk 和 monitor loop 会从辅助功能变成架构约束。

## 调制、探测与控制闭环

调制器是 Tx PIC 与 EIC 的交界面。常见结构包括 Mach-Zehnder modulator、microring modulator、electro-absorption modulator 和 TFLN MZM。

| 调制结构 | 机制 | 工程关注点 |
|---|---|---|
| MZM | 两臂相位差经干涉转换成强度变化 | 光学带宽、Vpi / VpiL、行波电极、长度、RF 损耗 |
| MRM | 谐振器 resonance 改变透过光强 | 面积小、能效潜力高，温度和工艺敏感，需要锁定 |
| EAM | 电场改变吸收 | 结构紧凑，偏置窗口、chirp、插损和光功率承受重要 |
| TFLN MZM | Pockels 效应改变折射率 | 高速、低损耗、线性好，平台和封装协同要求高 |

接收端 photodetector 把光功率转成电流，交给 TIA。高速 data PD 关注 responsivity、dark current、bandwidth、capacitance 和 saturation power；monitor PD 关注稳定性、线性和长期漂移。两类 PD 都属于 PIC / EIC 协同边界：PD 面积和 pad 会影响 TIA 噪声，monitor 的位置会影响控制环路判断。

控制闭环让 PIC 在温度和老化中维持工作点。heater、temperature sensor、monitor PD、bias DAC 和 firmware 会共同调 MZM bias、MRM resonance、AWG passband、WDM channel 和 optical power balance。控制余量也会成为运营信号：若同样输出需要更高 heater power 或更极端 bias，说明物理余量正在被消耗。

## PDK 设计流：从 building block 到版图

PIC 的 PDK 把可制造的光学单元、仿真模型、版图规则和测试结构提供给设计者。它的价值在于把“光学器件”变成可复用、可验证、可量产的 circuit building blocks。

| PDK 内容 | 对设计的意义 |
|---|---|
| waveguide templates | 规定层次、宽度、高度、弯曲半径、支持模式和 DRC 规则 |
| passive components | splitter、MMI、directional coupler、taper、grating、edge coupler、ring、AWG |
| active components | modulator、phase shifter、heater、photodetector、monitor tap 和电极 |
| compact models | 提供 S-parameter、transfer function、loss、phase、thermal tuning 和 bandwidth 模型 |
| layout / DFM rules | 管理 waveguide spacing、metal keepout、bump / pad、heater distance 和 test cells |

设计流要同时处理光学、电学、热学和封装几何。波导间距影响光串扰，电极影响 RF 损耗，heater 影响热串扰，microbump map 影响 EIC 摆放，coupler pitch 影响 FAU 和 fiber routing。PDK 的成熟度体现在模型、规则和测试数据能否持续互相校准。

## 测试与良率：PIC 必须可测

PIC 的良率由很多单元共同决定。一个 ring 偏移、一个 coupler 插损偏高、一个 heater 开路或一个 PD 暗电流异常，都可能让某条 lane 失效。设计阶段就要把 test structure、loopback、reference waveguide、monitor tap 和 KGD 策略放进版图。

| 测试层级 | 看到的内容 | 主要用途 |
|---|---|---|
| wafer-level optical test | coupler loss、waveguide loss、ring / AWG spectrum、splitter balance | 建立 wafer map，筛出工艺漂移和明显坏片 |
| wafer-level electrical / RF test | heater resistance、PD dark current、phase shifter IV、modulator S-parameter | 判断电极、pad、掺杂和高速结构是否在窗口内 |
| die sort / KGD | insertion-loss bin、wavelength bin、bandwidth bin、PD responsivity bin | 决定哪些 die 进入昂贵封装 |
| package-level link test | optical power、OMA、eye、TDECQ、BER、temperature sweep | 验证 laser、PIC、EIC、fiber 和热路径结合后的真实链路 |

PIC 的测试数据应能回到设计规则。若 wafer map 显示某个 reticle 区域的 AWG 中心波长偏移，问题可能来自线宽、厚度或应力；若 package test 显示某组 fiber attach 后插损长尾，问题可能在 optical I/O 和封装装配。成熟的 PIC 团队会把这些反馈变成版图 margin、工艺窗口和分档规则。

## PIC 交给封装和系统哪些条件

PIC 向相邻团队交付一组接口条件。

| PIC 侧条件 | 传给封装 / 系统的影响 |
|---|---|
| input optical power requirement | 决定 laser output、splitter plan、ELS path 和老化余量 |
| total insertion loss | 决定 Tx OMA、Rx sensitivity、laser heat 和 FEC margin |
| coupler type / pitch | 决定 FAU、lens、edge attach、grating access 和测试夹具 |
| pad / microbump map | 决定 EIC placement、RF path、PD-to-TIA parasitics 和 power delivery |
| heater power / thermal sensitivity | 决定散热设计、温控策略、启动校准和运行控制 |
| monitor signals | 决定 optical power、bias drift、wavelength drift 和 lane health 能否被追踪 |
| test coverage | 决定 KGD 可信度、package failure localization 和良率反馈 |

这些条件会互相牵制。缩短 EIC 到 modulator 的路径，会压缩 die placement；降低 coupler loss，会提高光学装配要求；提高 WDM 密度，会增加波长控制和 heater power；保留更多 monitor，会增加面积、光功率抽取和控制复杂度。PIC 的架构价值就在于把这些约束组织成可制造、可测试、可封装的光路。

## 总结：怎样读懂一张 PIC

读 PIC 时，先看架构，再看平台和器件库。架构回答 Tx / Rx、PSM / WDM、光功率和控制闭环怎样安排；平台回答哪些 building blocks 可用；PDK 和测试回答这张电路能否被制造和筛选；封装接口回答它能否进入真实 optical engine。
